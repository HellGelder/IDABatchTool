IDA Batch Tool

Кроссплатформенный инструмент для пакетного анализа бинарных файлов с использованием IDA Pro, Hex‑Rays и BinDiff.

* **Пакетный анализ** – параллельный запуск нескольких копий IDA.
* **Автоматический экспорт** – дизассемблер, псевдокод, метаданные в JSON.
* **HTML‑отчёты** – интерактивные отчёты с классификацией модулей, поиском и пояснениями метрик.
* **Сравнение директорий** – интеграция с BinDiff для построчного сравнения функций.
* **Поддержка архивов** – автоматическая распаковка APK, IPA, DMG.
* **Графический интерфейс** – современный UI на PySide6 (светлая/тёмная темы).
* **CLI** – возможность запуска из командной строки для автоматизации.

#Системные требования

* **Python 3.10+**
* **IDA Pro 9.x** с плагинами **BinExport 12** и **BinDiff 8** (для сравнения)
* **7-Zip** (только для извлечения `.dmg` на Windows/Linux)
* **Операционная система:** Windows 10/11, Linux (Debian/Ubuntu), macOS 12+

#Установка

##1. Клонирование репозитория и установка зависимостей Python

```bash
git clone <repository_url>
cd IDABatchTool
pip install -r requirements.txt
```

##2. Настройка IDA Pro

Убедитесь, что в папке `plugins` вашей IDA Pro установлен **BinExport 12** (файл `binexport12_ida64.dll` или аналогичный для вашей ОС).  
При запуске IDA в консоли вы должны увидеть строку:

```
BinExport 12 (@6447866, Mar  2 2026), (c)2004-2011 zynamics GmbH, (c)2011-2025 Google LLC.
```

Если плагин не загружается, проверьте пути в `config.yaml` (см. [Конфигурация](#конфигурация)).

##3. Установка BinDiff (для сравнения директорий)

BinDiff используется как отдельная программа, вызываемая из командной строки.

###Windows

1. Скачайте [bindiff_8_amd64.msi](https://github.com/google/bindiff/releases/tag/v8).
2. Установите MSI, следуя инструкциям.
3. Убедитесь, что `bindiff.exe` доступен из PATH, или скопируйте его в корень проекта.

###Linux (Debian/Ubuntu)

```bash
wget https://github.com/google/bindiff/releases/download/v8/bindiff_8_amd64.deb
sudo dpkg -i bindiff_8_amd64.deb
Если возникнут проблемы с зависимостями:
sudo apt --fix-broken install
```

###macOS

```bash
brew install --cask bindiff
или скачайте .dmg с https://github.com/google/bindiff/releases/tag/v8
```

После установки `bindiff` должен быть доступен в терминале.

##4. Установка 7-Zip (для распаковки DMG)

###Windows
[Скачайте 7-Zip](https://www.7-zip.org/) и убедитесь, что `7z.exe` доступен из PATH.

###Linux
```bash
sudo apt install p7zip-full
```

###macOS
```bash
brew install p7zip
```

Приложение проверит наличие `7z` при обнаружении `.dmg` и предупредит, если он не найден.

#Быстрый старт (CLI)

```bash
Анализ всех .exe и .dll в папке с последующим экспортом в JSON и созданием HTML-отчёта
python main.py --inputdir ./samples --analyse --filter ".exe,.dll" --report
```

Основные опции CLI:

| Флаг | Описание |
|------|----------|
| `--inputdir` | Папка с бинарными файлами |
| `--analyse` | Запустить автоанализ и создать базы `.i64` |
| `--filter` | Фильтр расширений через запятую (по умолчанию `.exe,.dll,.elf,.so`) |
| `--max-ida` | Максимальное число параллельных копий IDA |
| `--idat` | Путь к исполняемому файлу IDA |
| `--script` | Дополнительный IDAPython-скрипт |
| `--report` | Создать HTML-отчёты |

#Графический интерфейс

Запуск GUI:

```bash
python main_gui.py
```

##Страницы

1. **Анализ** — выбор директории, настройка платформы, флагов анализа, запуск. После завершения доступна кнопка «Сгенерировать HTML-отчёты».
2. **Сравнение** — выбор двух директорий с `.i64`, автоматическое сопоставление, запуск сравнения и генерация детальных отчётов.
3. **Конфигурация** — пути к IDA и BinDiff, выбор темы оформления.

##Темы
Поддерживаются светлая и тёмная темы, переключаемые на странице «Конфигурация».

#Сравнение директорий (BinDiff)

Страница «Сравнение» позволяет сопоставить две версии одного приложения.

1. Укажите левую (эталонную) и правую (текущую) папки с базами `.i64`.
2. Виджет сопоставления покажет, зеркальные ли директории, и выделит отсутствующие файлы.
3. Нажмите «Запустить сравнение» — для каждой пары будет выполнен экспорт в `.BinExport`, запущен `bindiff`, результат распарсен.
4. После завершения нажмите «Сгенерировать HTML-отчёт» — в папке `Reports` появится индексный файл со сводной статистикой и ссылками на детальные отчёты.

Метрики сравнения (с подробными подсказками в интерфейсе):
- **Схожесть (Similarity)** — структурная близость файлов/функций.
- **Уверенность (Confidence)** — надёжность найденного соответствия.
- **Совпадающие базовые блоки, инструкции, рёбра** — степень детального совпадения.

#Работа с архивами

Инструмент умеет извлекать исполняемые файлы из:

- **APK** (Android) — библиотеки `.so` из папки `lib/`
- **IPA** (iOS) — исполняемый файл приложения из `Payload/*.app/`
- **DMG** (macOS) — содержимое образа (требуется `7z`)

Достаточно положить архив в анализируемую папку. Архив будет автоматически распакован в соседнюю папку с тем же именем, и все исполняемые файлы внутри него добавятся в очередь анализа.

#Конфигурация

Файл `config.yaml` в корне проекта управляет настройками.

```yaml
ida:
  executable: idat         путь к idat/idat.exe (можно полный)
bindiff:
  executable: bindiff      путь к bindiff/bindiff.exe
max_ida: 6                 максимальное число параллельных копий IDA
default_inputdir: .        директория по умолчанию для анализа
log_level: INFO            DEBUG, INFO, WARNING, ERROR
theme: dark                light или dark
```

Настройки можно изменить также через страницу «Конфигурация» в GUI.
