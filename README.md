# IDA Batch Tool

Кроссплатформенный инструмент для пакетного анализа бинарных файлов с использованием IDA Pro, Hex‑Rays и BinDiff.

* **Пакетный анализ** — параллельный запуск нескольких копий IDA.
* **Автоматический экспорт** — дизассемблер, псевдокод, метаданные в JSON.
* **HTML‑отчёты в стиле Apple** — интерактивные отчёты с групповым списком категорий, drill‑down до модулей, быстрым поиском, подробными тултипами метрик и прогресс-барами.
* **Сравнение директорий** — интеграция с BinDiff для построчного сравнения функций.
* **Анализ системных функций (СФ)** — документация Win32 API с Markdown-рендерингом и вкладками.
* **Поддержка архивов** — автоматическая распаковка APK, IPA, DMG.
* **Графический интерфейс** — современный UI на PySide6.
* **CLI** — возможность запуска из командной строки для автоматизации.

# Установка

## 1. Клонирование репозитория и зависимости Python

```bash
git clone https://github.com/<your-org>/IDABatchTool.git
cd IDABatchTool
pip install -r requirements.txt
```

Файл `requirements.txt` содержит закреплённые минимальные версии:

```
pyyaml>=6.0

beautifulsoup4>=4.11

gitpython>=3.1
pyelftools>=0.29

circlify>=0.10
markdown>=3.4
```

## 2. IDA Pro 9.x

Установите IDA Pro версии **9.0 или выше**. Инструмент использует текстовый интерфейс IDA (`idat` / `idat.exe`) в пакетном режиме — GUI не требуется.

После установки убедитесь, что `idat` доступен одним из способов:
- Прописан в системном `PATH`
- Указан абсолютный путь в `config.yaml` (ключ `ida.executable`)
- Находится в типичной папке установки:
  - Windows: `C:\Program Files\IDA Professional 9.x\idat.exe`
  - Linux: `~/ida-pro-9.0/idat` или `/opt/ida-pro-9.0/idat`
  - macOS: `/Applications/IDA Professional 9.x.app/Contents/MacOS/idat`

Инструмент автоматически ищет IDA в этих путях, если в конфиге указано просто `idat`.

## 3. Плагин BinExport 12

**Обязателен.** Используется для экспорта баз `.i64` в формат `.BinExport`, необходимый для сравнения в BinDiff.

1. Скачайте [BinExport 12](https://github.com/google/binexport/releases/tag/v12) для вашей ОС и архитектуры IDA.
2. Скопируйте файл плагина в папку `plugins` вашей установки IDA Pro:
   - Windows: `C:\Program Files\IDA Professional 9.x\plugins\binexport12_ida64.dll`
   - Linux: `~/ida-pro-9.0/plugins/binexport12_ida64.so`
   - macOS: `~/ida-pro-9.0/plugins/binexport12_ida64.dylib`
3. При запуске IDA в консоли убедитесь, что плагин загружается:
   ```
   BinExport 12 (@6447866, Mar  2 2026), (c)2004-2011 zynamics GmbH, (c)2011-2025 Google LLC.
   ```

**Важно:** Инструмент вызывает BinExport через headless-флаг IDA: `-OBinExportAutoAction:BinExportBinary`. Плагин должен поддерживать `AutoAction` — BinExport 12 поддерживает.

## 4. BinDiff 8 (standalone CLI)

**Обязателен для сравнения директорий.** Используется standalone-версия `bindiff` (командная строка), **не** IDA-плагин.

> **Почему standalone, а не плагин?**  
> Плагин BinDiff для IDA спроектирован исключительно для интерактивного GUI. Он не имеет механизма `AutoAction` (в отличие от BinExport) и требует GUI-диалогов выбора файлов. Для headless-пайплайна разработчики Google предоставляют отдельный CLI-инструмент `bindiff.exe` / `bindiff`, который и используется в проекте. Это не обходной путь — это документированный Best Practice.

### Windows

1. Скачайте [bindiff_8_amd64.msi](https://github.com/google/bindiff/releases/tag/v8).
2. Установите MSI, следуя инструкциям.
3. Убедитесь, что `bindiff.exe` доступен из `PATH`, или скопируйте его в корень проекта, или укажите полный путь в `config.yaml` (ключ `bindiff.executable`).

### Linux (Debian/Ubuntu)

```bash
wget https://github.com/google/bindiff/releases/download/v8/bindiff_8_amd64.deb
sudo dpkg -i bindiff_8_amd64.deb
# Если возникнут проблемы с зависимостями:
sudo apt --fix-broken install
```

### macOS

```bash
brew install --cask bindiff
# или скачайте .dmg с https://github.com/google/bindiff/releases/tag/v8
```

После установки `bindiff` должен быть доступен в терминале:
```bash
bindiff --help
```

## 5. Плагин BinDiff 8 (опционально, для GUI IDA)

Если вы также хотите использовать BinDiff **интерактивно** внутри IDA Pro (вне данного инструмента), установите плагин:

1. Скачайте и установите BinDiff 8 (тот же установщик, что и для CLI).
2. При установке выберите интеграцию с IDA Pro.
3. Файлы плагина (`bindiff8_ida64.dll` и др.) будут помещены в папку `plugins` вашей IDA.

На работу данного инструмента это не влияет — он всегда использует standalone CLI.

# Быстрый старт (CLI)

Анализ всех `.exe` и `.dll` в папке с экспортом в JSON и генерацией HTML-отчёта:

```bash
python main.py --inputdir ./samples --analyse --filter ".exe,.dll" --report
```

Генерация HTML-отчётов по уже существующим `.i64` / `.export.json` (без повторного анализа):

```bash
python main.py --inputdir ./samples --report
```

Основные опции CLI:

| Флаг | Описание |
|------|----------|
| `--inputdir` | Папка с бинарными файлами (обязательно) |
| `--analyse` | Запустить автоанализ и создать базы `.i64` |
| `--filter` | Фильтр расширений через запятую (по умолчанию `.exe,.dll,.elf,.so`) |
| `--max-ida` | Максимальное число параллельных копий IDA |
| `--idat` | Путь к исполняемому файлу IDA |
| `--script` | Дополнительный IDAPython-скрипт |
| `--report` | Создать HTML-отчёты из `.i64` / `.export.json` |
| `--verbose` | Подробный вывод (DEBUG) |

## Страницы

1. **Анализ** — выбор директории, автоопределение платформы по расширениям, настройка флагов анализа, прогресс-бар и Treemap статуса файлов.

# HTML-отчёты

HTML-отчёты спроектированы в духе Apple Human Interface Guidelines:

- **Сводный отчёт анализа** (`IDAReports/index.html`) — hero-карточка с общей информацией, плитки статистики, **интерактивный список категорий импортированных модулей с drill-down** в правую панель, список секций ELF, перечень исследованных файлов со ссылками.
- **Индивидуальный отчёт** (`*.html`) — sticky-заголовок, поиск по функциям/импортам/экспортам, аккордеоны с зависимостями (цветные маркеры, grid-сетка), таблицы импортов и экспортов, дизассемблированные функции с hex-дампом и псевдокодом.
- **Сводка сравнения** (`DiffResults/Reports/index.html`) — статистика по парам, таблица с tinted-бейджами схожести (зелёный >80%, жёлтый 50-80%, красный <50%), статусами хэшей.
- **Отчёт сравнения пары** (`*.html`) — прогресс-бары схожести/уверенности, grouped lists статистики по каждому файлу, таблица совпавших функций с тултипами методики.
- **Сводка СФ** (`SFAReports/index.html`) — плитки статистики и список отчётов.
- **Детальный СФ** (`*.sfa.html`) — split-view: sidebar со списком системных вызовов + интерактивная панель с параметрами и вкладками документации (Markdown).

Все отчёты полностью автономны (offline): JavaScript-зависимости (`marked.min.js` для Markdown, `chart.umd.min.js` для графиков) вендорятся локально.

# Сравнение директорий (BinDiff)

Страница «Сравнение» позволяет сопоставить две версии одного приложения.

## Принцип работы

1. **Экспорт через плагин BinExport (headless IDA)**:
   ```
   idat -A -OBinExportAutoAction:BinExportBinary -OBinExportModule:<output.BinExport> <file.i64>
   ```
   Плагин BinExport поддерживает `AutoAction` — выполняет экспорт и завершает IDA автоматически без GUI.

2. **Сравнение через standalone bindiff CLI**:
   ```
   bindiff --primary <file1.BinExport> --secondary <file2.BinExport> --output_dir <tmp>
   ```
   Результат — SQLite-база `.BinDiff`.

3. **Парсинг** `.BinDiff` → `.diff.json` с метаданными, совпавшими функциями и метриками.

4. **Генерация HTML-отчётов** с прогресс-барами, таблицами и тултипами методики.

## Пошаговая инструкция

1. Укажите левую (эталонную) и правую (текущую) папки с базами `.i64`.
2. Виджет сопоставления покажет, зеркальные ли директории, и выделит отсутствующие файлы.
3. Нажмите «Запустить сравнение» — в фоновом потоке выполнится экспорт + сравнение для каждой пары.
4. После завершения нажмите «Сгенерировать HTML-отчёт» — в папке `DiffResults/Reports` появится индексный файл со сводной статистикой и ссылками на детальные отчёты.

## Метрики сравнения

| Метрика | Описание |
|---------|----------|
| **Схожесть (Similarity)** | 0–1. Структурная близость файлов. Формула: ~35% рёбра потока управления, ~25% базовые блоки, ~10% функции, ~10% инструкции, ~20% разница MD-индекса графа вызовов |
| **Уверенность (Confidence)** | 0–1. Надёжность найденного соответствия. Среднее качество алгоритмов, сглаженное сигмоидой |
| **Базовые блоки** | Последовательности инструкций с одной точкой входа и выхода — минимальные единицы сравнения |
| **Инструкции** | Сравниваются по мнемоникам (операнды игнорируются — адреса и константы могут меняться) |
| **Рёбра CFG** | Связи между базовыми блоками в графе потока управления |

# Анализ системных функций (СФ)

Страница «Анализ СФ» документирует вызовы Windows API в бинарных файлах.

1. Укажите директорию с бинарными файлами (аналогично странице «Анализ»).
2. Нажмите «Запустить анализ СФ» — выполнится экспорт в JSON (как на странице «Анализ»).
3. Нажмите «Сгенерировать HTML-отчёт СФ».
4. При первом запуске инструмент предложит синхронизировать базу сигнатур Win32 API из GitHub-репозитория.
5. Для каждой функции производится поиск документации через `npx @microsoft/learn-cli search`. Результаты кэшируются в `doc_cache.json`.

# Работа с архивами

Инструмент умеет извлекать исполняемые файлы из:

- **APK** (Android) — библиотеки `.so` из `lib/`
- **IPA** (iOS) — исполняемый файл приложения из `Payload/*.app/`
- **DMG** (macOS) — содержимое образа (требуется `7z`)

Достаточно положить архив в анализируемую папку. Архив будет автоматически распакован в соседнюю папку с тем же именем, и все исполняемые файлы внутри него добавятся в очередь анализа. Повторная распаковка не производится — если папка уже существует, она используется как есть.

# Конфигурация

Файл `config.yaml` в корне проекта управляет настройками. При первом запуске создайте его из примера:

```bash
cp config.yaml.example config.yaml
```

Содержимое `config.yaml.example`:

```yaml
# Путь к исполняемому файлу IDA Pro 9.0+ (idat).
ida:
  executable: idat

# Путь к исполняемому файлу BinDiff (standalone CLI).
bindiff:
  executable: bindiff

# Каталог для анализа по умолчанию.
default_inputdir: .

# Максимальное количество параллельных экземпляров IDA.
max_ida: 4

# Уровень логирования: DEBUG / INFO / WARNING / ERROR.
log_level: INFO

# Тема интерфейса: light / dark.
theme: light

# Путь к папке хранения локальных БД сигнатур (win32api.db).
sf_db_path: databases
```

Настройки можно изменить также через страницу «Конфигурация» в GUI.

> **Важно:** `config.yaml` добавлен в `.gitignore`. Не коммитьте его в репозиторий — он содержит машинно-зависимые абсолютные пути.

# FAQ

### Почему используется standalone bindiff.exe, а не плагин BinDiff для IDA?

Плагин BinDiff для IDA спроектирован исключительно для интерактивного GUI. У него нет механизма `AutoAction` (в отличие от BinExport), и внутренние функции используют GUI-диалоги выбора файлов, которые зависают в headless-режиме (`-A`). Для автоматизированного пайплайна Google предоставляет отдельный CLI-инструмент `bindiff`, который и используется в проекте.

### Почему HTML-отчёты не поддерживают тёмную тему?

Отчёты используют единый CSS-partial с Apple-палитрой (светлая). Тёмная тема для отчётов не реализована, так как они предназначены для чтения и архивации. GUI поддерживает обе темы.

### Можно ли анализировать файлы без интернета?

Да. Все зависимости HTML-отчётов (`marked.min.js`, `chart.umd.min.js`) вендорены локально в `templates/vendor/`. Единственная функция, требующая интернета — поиск документации СФ через `npx @microsoft/learn-cli search`. Результаты поиска кэшируются в `doc_cache.json`.

### Как добавить поддержку новой платформы в классификатор?

Добавьте словарь модулей (как в `classifier/android.py`) и зарегистрируйте его в `classifier/platform_classifier.py`. Формат: `{ "имя_модуля": "описание на русском" }`.
