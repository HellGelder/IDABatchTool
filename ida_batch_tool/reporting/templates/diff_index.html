Описание метрик сравнения

Ниже приведено описание метрик, используемых движками сравнения для оценки схожести бинарных файлов.

BinDiff (Google)

Схожесть (Similarity) — значение от 0 до 1, отражающее общую структурную близость двух файлов. Рассчитывается как взвешенная комбинация: ~35% — доля совпавших рёбер потока управления, ~25% — доля совпавших базовых блоков, ~10% — доля совпавших функций, ~10% — доля совпавших инструкций, ~20% — разница индекса MD графа вызовов. Чем ближе к 1.0, тем больше общих структурных элементов.

Уверенность (Confidence) — значение от 0 до 1, характеризующее надёжность найденного соответствия. Рассчитывается как среднее качество сработавших алгоритмов сопоставления, взвешенное сигмоидной функцией. Высокая схожесть при низкой уверенности означает, что слабый алгоритм дал хороший результат; низкая схожесть при высокой уверенности — сильный алгоритм не обнаружил сходства.

19 алгоритмов матчинга BinDiff (официальная документация: zynamics.com/bindiff/manual):

Развернуть список
  1. Same function hash — совпадение хэша функции (SHA-1 от сырых байтов, нормализованных по смещениям);
  2. Same name hash — совпадение хэша имени функции (для именованных функций);
  3. Same call reference — совпадение по ссылкам вызова (функции, на которые ссылается одинаковый набор вызывающих);
  4. Same address — совпадение по адресу (если адрес функции одинаков в обоих файлах);
  5. Same bytes hash — совпадение хэша байтового содержимого функции;
  6. Same MD-Index (structural hash) — совпадение MD-индекса (структурного хэша графа потока управления);
  7. Same string reference — совпадение по строкам, на которые ссылается функция;
  8. Same constant reference — совпадение по непосредственным значениям (константам);
  9. Same instruction mnemonics — совпадение мнемоник инструкций;
  10. Same basic blocks count — совпадение количества базовых блоков;
  11. Same instructions count — совпадение количества инструкций;
  12. Same edges count — совпадение количества рёбер графа;
  13. Same call count — совпадение количества вызовов внутри функции;
  14. Same indegree/outdegree — совпадение входящей/исходящей степени в графе вызовов;
  15. Same name (symbolic) — совпадение символического имени через таблицу экспорта/отладки;
  16. Call reference matching (secondary) — совпадение по ссылкам на другие сопоставленные функции;
  17. Same basic block hash — совпадение хэшей базовых блоков внутри функции;
  18. Same MD-Index (relaxed) — смягчённое совпадение MD-индекса (с допуском по структуре);
  19. Same name (import/export table) — совпадение по таблице импорта/экспорта.

При сопоставлении по имени BinDiff проверяет не только совпадение символического имени, но и внутреннюю структуру функции (хэш тела функции, количество базовых блоков, инструкций и рёбер). Если структура совпадает — алгоритм «name hash matching». Если структура различается — может сработать другой алгоритм или совпадение не засчитывается.

Diaphora (Joxean Koret)

Схожесть (Ratio) — значение от 0 до 1, вычисленное по результатам сработавшей эвристики. Каждая эвристика имеет свой порог минимального ratio. Чем выше ratio, тем надёжнее совпадение.

Типы совпадений:

  • Best — наивысшая надёжность, эвристики не дают ложных срабатываний;
  • Partial — средняя надёжность, возможны ложные срабатывания;
  • Unreliable — низкая надёжность, только как подсказка.

Пофункциональный разбор: Для каждой пары функций (со схожестью менее 100%) выполняется: сравнение псевдокода (side-by-side diff), сравнение hex-дампа, граф потока управления (CFG), расхождения по типам инструкций.

Дополнительные метрики

Hex-dump сравнение: Полное поблочное (16 байт) side-by-side сравнение исходных исполняемых файлов. При идентичности показываются первые 4 строки, многоточие и последние 4 строки. При расхождениях — все строки с подсветкой различий.
