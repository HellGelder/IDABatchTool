Описание метрик сравнения

Ниже приведено описание метрик, используемых движками сравнения для оценки схожести бинарных файлов.

BinDiff (Google)

Схожесть (Similarity) — значение от 0 до 1, отражающее общую структурную близость двух файлов. Рассчитывается как взвешенная комбинация: ~35% — доля совпавших рёбер потока управления, ~25% — доля совпавших базовых блоков, ~10% — доля совпавших функций, ~10% — доля совпавших инструкций, ~20% — разница индекса MD графа вызовов. Чем ближе к 1.0, тем больше общих структурных элементов.

Уверенность (Confidence) — значение от 0 до 1, характеризующее надёжность найденного соответствия. Рассчитывается как среднее качество сработавших алгоритмов сопоставления, взвешенное сигмоидной функцией. Высокая схожесть при низкой уверенности означает, что слабый алгоритм дал хороший результат; низкая схожесть при высокой уверенности — сильный алгоритм не обнаружил сходства.

Алгоритмы матчинга BinDiff (19 шт.): BinDiff использует 19 эвристик для сопоставления функций, включая: хэш имени функции, хэш сырых байтов, совпадение по адресу, совпадение по вызовам (call reference matching), MD-индекс (структурный хэш графа потока управления), совпадение по мнемоникам и другие. Каждая пара функций маркируется алгоритмом, который дал наилучшее совпадение.

При сопоставлении по имени BinDiff проверяет не только совпадение символического имени, но и внутреннюю структуру функции (хэш тела функции, количество базовых блоков, инструкций и рёбер). Если структура совпадает — алгоритм «name hash matching». Если структура различается — может сработать другой алгоритм или совпадение не засчитывается.

Diaphora (Joxean Koret)

Схожесть (Ratio) — значение от 0 до 1, вычисленное по результатам сработавшей эвристики. Каждая эвристика имеет свой порог минимального ratio. Чем выше ratio, тем надёжнее совпадение.

Типы совпадений:

  • Best — наивысшая надёжность, эвристики не дают ложных срабатываний (хэш функции, байтовый хэш, идентичный ассемблер/псевдокод);
  • Partial — средняя надёжность, возможны ложные срабатывания (совпадение констант, нечёткий хэш псевдокода, switch-структуры);
  • Unreliable — низкая надёжность, только как подсказка (совпадение графа, топологическая сортировка, высокая сложность).

Эвристики Diaphora (50 шт.): Diaphora использует 50 различных эвристик, включая: совпадение RVA + хэш, хэш функции, байтовый хэш, совпадение адреса и мнемоник, нечёткий хэш псевдокода (нормальный/обратный/смешанный/AST), хэш KOKA, MD-индекс, совпадение констант, структуры switch/case, количество циклов, единица компиляции, хэши импортов и многие другие. Полный список выводится в детальном отчёте.

Пофункциональный разбор: Для каждой пары функций (со схожестью менее 100%) выполняется: сравнение псевдокода (side-by-side diff), сравнение hex-дампа (с пропуском идентичных блоков), граф потока управления (CFG), расхождения по типам инструкций.

Дополнительные метрики

Hex-dump сравнение: Попарное сравнение исходных исполняемых файлов поблочно (16 байт). Идентичные блоки пропускаются (как Linux hexdump). Выводится в детальном отчёте в блоке «Сравнение hex-дампов».

Hex-схожесть (SHA256): Сравнение бинарных файлов по хэшам SHA256 блоков по 4096 байт. Показывает долю совпавших блоков с учётом соотношения размеров файлов.
